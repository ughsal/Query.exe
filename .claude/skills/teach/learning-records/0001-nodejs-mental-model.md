# Learning Record 0001: Node.js Mental Model

**Date:** 2026-06-25
**Topic:** The fundamental difference between Client-side (Browser) and Server-side (Node.js) execution.

## Key Insights
- User understands that Node.js is a runtime, not a language.
- User recognizes the Request-Response cycle as the primary driver of backend logic.
- User can map the concept of "listening on a port" to how a server waits for requests, contrasting it with the browser's event-driven UI updates.

## ZPD Assessment
The user has a strong TS/JS foundation, so the syntax is not a barrier. The primary "gap" is the conceptual shift to persistent server processes and the stateless nature of HTTP requests. They are ready to move into the specifics of Express middleware.
